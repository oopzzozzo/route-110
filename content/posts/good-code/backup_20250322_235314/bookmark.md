因為這陣子事情很多，就忍不住開始整理東西。
先從我四年來一直很混亂的 firefox bookmark 下手。

基本上我沒有固定追蹤部落格之類，所以 bookmark 真的就是當書籤，全塞 Others。
因為書籤很亂，所以工具列沒有書籤列，只有囤積很久，越消越多的 tab。
同時我還用了 Tab Session Manager 和 Pocket，當然我手機也是用 FF。
Tab Session Manager 是因為以前會點各種論文，卻一篇都沒讀，就先堆一個 Session。
Pocket 單純是因為它在工具列就按過。後來才發現開 tab + bookmark 全同步比較乾脆。

我起手勢把大三以來的書籤逐一點開，檢查還在不在，並分類。
可想而知分類不容易。每每掙扎該不該多建一類時，就得說服自己先做一輪再來想。
做著做著我心想：「技術相關的文章，到底該依領域分類還是抽象程度分類呢？」
例如說：virt-manager tutorial, mongoDB manual, IO virtualization, graph DB
我在查找分類時，哪個維度分類比較明確？瀏𣞢時又在哪個維度比較有 locality？

想著想著：「現在都 21 世紀了，應該要上 tag 列舉顯示而不是弄成樹狀結構吧？」
定睛一看，原來我分類當下用的書籤管理頁面，點開一書籤，內容 URL 下面就是 tags…
除了 tags 還有 keyword，作用相當於 alias。
我真是低能使用者耶，整個功能擺在我眼前半個多小時我竟然看不到。
